# 声绘课堂 — 技术架构文档

## 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 桌面框架 | Electron 33 | CS 架构，本地文件读写、系统 API |
| 构建工具 | electron-vite 3 + Vite 6 | 统一管理三进程构建 |
| 前端框架 | React 18 | 函数组件 + Hooks |
| 类型系统 | TypeScript 5 | 全项目强类型 |
| UI 样式 | Tailwind CSS 3 | 工具类样式，暗色主题 |
| 图标库 | lucide-react | SVG 图标 |
| AI 接口 | Google Gemini API | LLM 评价生成（可配置） |
| ASR 接口 | Whisper V3 (OpenAI 兼容) | 带 Speaker 标签的语音转写 |
| 推送通道 | 企业微信外部联系人 API | 家长消息推送 |
| 存储方案 | 本地文件系统（免数据库） | JSON + Markdown + WAV |

## 数据流

```
录音(MediaRecorder) → 分段PCM → 主进程写入 segments/
       ↓
Whisper V3 ASR → 课程名_ASR.md
       ↓
Gemini LLM → 课程名_Analysis.md（支持迭代修改）
       ↓
人工核对 → 企业微信推送 → _Status.json 记录状态
```

## 本地存储结构

```
[用户配置根目录]/
└── 2026年02月27日/
    └── 英语课_1班_0900/
        ├── 英语课_1班_0900.wav        # 原始录音
        ├── 英语课_1班_0900_ASR.md     # ASR 转写
        ├── 英语课_1班_0900_Analysis.md # LLM 评价
        ├── _Status.json               # 状态持久化
        └── segments/                  # 流式 ASR 临时切片
```

## 核心类型定义

```typescript
// 四阶段流水线状态
type StageStatus = 'pending' | 'processing' | 'done' | 'error'

// 课程记录
interface Course {
  id: string          // 课程名_班级名_开始时间
  date: string        // 如：2026年02月27日
  pipeline: { rec, asr, analysis, push: StageStatus }
  students: StudentRecord[]
  asrText?: string
  analysisText?: string
}

// 学生推送记录
interface StudentRecord {
  pushStatus: 'pending' | 'sent' | 'failed' | 'missing_id'
}
```
